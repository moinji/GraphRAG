1. 온톨로지 품질 메트릭 (evaluator.py + quality_checker.py)
   - critical_error_rate: 누락 노드/관계 비율 (0.0~1.0)
   - fk_relationship_coverage: 골든 데이터 대비 관계 매칭률 (0.0~1.0)
   - direction_accuracy: 관계 방향 정확도 (0.0~1.0)
   - quality_score: 1.0 - (0.2×에러수 + 0.05×경고수), 최소 0.0
   - 5가지 Protégé 기준 검증: 중복 클래스, Domain/Range 오류, 순환 참조, 고아 노드, 네이밍 규칙

2. Q&A 평가 메트릭 (qa_evaluator.py)
   - keyword_score: 기대 키워드 매칭률 (0.0~1.0)
   - entity_score: 기대 엔티티 매칭률 (0.0~1.0)
   - success: keyword_score > 0이면 True
   - 50개 골든 QA 페어 (이커머스): traverse 20, aggregate 10, comparison 8, complex 5, unsupported 5
   - 15개 골든 QA 페어 (비정형 문서용, Mode C)
   - A/B 비교 리포트: 성공수, 평균 레이턴시, 평균 키워드 점수, 토큰 수, 비용

3. 런타임 메트릭 (pipeline.py, hybrid_search.py, local_search.py)
   - latency_ms: 요청→응답 전체 시간 (ms)
   - llm_tokens_used: LLM 호출당 토큰 수
   - cached: 데모 캐시 히트 여부

4. LLM 비용 추적 (llm_tracker.py)
   - 호출별: caller, model, input/output tokens, estimated_cost_usd
   - 모델별 요율: GPT-4o ($2.50/$10.00), Sonnet ($3.00/$15.00), Haiku ($0.80/$4.00)
   - API: GET /api/v1/llm-usage (전체 요약), DELETE (리셋)

5. 인프라 헬스 (health.py)
   - neo4j.latency_ms, postgres.latency_ms: 연결 확인 시간
   - status: healthy | degraded

성능 관련:
- FK 규칙 엔진: 결정적, 즉시 (~100ms 이내)
- LLM 보강 (Stage 2): Claude Sonnet, 보통 5~20초
- Neo4j 로드: UNWIND + MERGE, 배치 5000건 제한
- 벡터 검색: pgvector HNSW (cosine), PostgreSQL 16 동일 DB
- Mode A (Cypher 템플릿): LLM 불필요, 즉시
- Mode B (로컬 서치): 서브그래프 추출 + LLM 요약, 2~8초
- Mode C (하이브리드): 벡터 검색 + KG 컨텍스트 + LLM 합성, 3~10초
- 데모 캐시: 5개 질문 100% 즉시 응답

규모 관련:
- PoC 데모 데이터: 12 테이블, ~100행 (이커머스)
- 교육 시드: 11 테이블, 73행
- 보험 시드: 12 테이블, 70행
- CSV 최대: 50MB, 50개 파일
- RDB 직접 연결: 미구현 (CSV 업로드로 대체)
- 대규모 테스트 (100만행): 미실시

비용 관련:
- Blended LLM cost: ~$6.30 / 1M tokens (Sonnet 기준)
- 온톨로지 생성 1회: ~2,000~5,000 tokens
- Q&A 1회 (Mode B): ~1,000~3,000 tokens
- Q&A 1회 (Mode C): ~1,500~4,000 tokens
- Mode A: 0 tokens (LLM 불필요)

7. PoC 단계 vs MVP 단계 vs Production 단계를 구분하여 단계별 목표 제시

## 사용법

1. 위 프롬프트 전체를 Claude / GPT-4에 붙여넣기
2. 프로젝트 상황이 변경되면 `[실측 데이터]` 섹션만 업데이트
3. 출력된 KPI 테이블을 투자자 발표자료·내부 대시보드에 직접 활용
