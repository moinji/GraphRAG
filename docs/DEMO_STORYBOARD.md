**멘트**: "RDB를 건드리지 않고, DDL 하나로 관계 추론 Q&A를 만듭니다"

### [0:15─0:45] Act 1 — DDL → 온톨로지 (Auto Mode)
**화면**: DDL 업로드 페이지
**동작**:
1. `demo_ecommerce.sql` (12테이블) 드래그 앤 드롭
2. 파싱 완료 → ERD 요약 표시 (테이블 12개, FK 13개)
3. FK 규칙 엔진 → 온톨로지 자동 생성 (노드 9개, 관계 12개)
4. Auto Mode → 자동 승인

**핵심 메시지**: "FK에서 자동으로 온톨로지가 만들어집니다. 10분 안에 첫 질문 답변이 가능합니다."

### [1:30─2:40] Act 4 — Q&A 데모 (핵심!)
**화면**: Q&A 채팅 인터페이스 (또는 터미널)

#### Q1 (0:20) — 2홉 관계 추론
```
질문: "고객 김민수가 주문한 상품 목록을 보여줘"
답변: 맥북 프로, 에어팟 프로, 갤럭시 탭
근거: Customer(김민수) → PLACED → Order(#1024) → CONTAINS → Product(맥북 프로)
```
**포인트**: 단순 조회가 아닌 **관계 경로**가 근거로 출력됨

#### Q2 (0:25) — 3홉 + 집계
```
질문: "김민수가 산 상품과 같은 카테고리의 리뷰 평점 Top 3는?"
답변: 1위 갤럭시 S24(4.8), 2위 LG 그램(4.6), 3위 소니 WH-1000(4.5)
근거: Customer→Order→Product→BELONGS_TO→Category→Product→Review
```
**포인트**: **다단계 관계 추론** — RDB SQL로는 복잡한 조인이 필요한 질문

#### Q3 (0:25) — 집계 + 근거
```
질문: "가장 많이 팔린 카테고리 Top 3와 대표 상품은?"
답변: 1위 전자기기(142건), 2위 의류(98건), 3위 식품(76건)
근거: Cypher 집계 쿼리 + 대표 경로(Order→Product→Category)
```
**포인트**: **집계도 그래프에서 근거와 함께** 제공

**기능 범위 (개발 필요)**:
- [x] Cypher 템플릿 3개 (two_hop, three_hop+top_n, agg_with_rel)
- [x] 슬롯 채우기 + Cypher 실행
- [x] 경로(paths) 추출 및 포맷팅
- [ ] 채팅 UI — v0에서는 CLI/API 출력으로 대체
