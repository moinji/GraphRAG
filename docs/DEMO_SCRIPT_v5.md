# GraphRAG v5.0 데모 스크립트 — 정형 + 비정형 통합 Q&A

> 목표 시간: 4분 | 대상: 기술 투자자, B2B 고객
> 핵심 메시지: "하나의 시스템에서 정형 데이터와 비정형 문서를 모두 검색·질의할 수 있습니다."

---

## 사전 준비

```bash
# 1. 서비스 시작
docker compose up -d
cd backend && uvicorn app.main:application --port 8080
cd frontend && npm run dev

# 2. 브라우저에서 http://localhost:5173 접속
```

필요 파일:
- `examples/schemas/demo_ecommerce.sql` (DDL)
- `examples/documents/맥북프로_제품설명.md`
- `examples/documents/에어팟프로_제품설명.md`
- `examples/documents/쇼핑몰_이용안내.md`

---

## Act 1: 정형 데이터 파이프라인 (1분)

### 1-1. DDL 업로드 (15초)
> "이커머스 DDL을 업로드하면 자동으로 온톨로지가 생성됩니다."

1. UploadPage → `demo_ecommerce.sql` 드래그 앤 드롭
2. "생성" 클릭 → 온톨로지 결과 확인 (9 노드, 12 관계)
3. Auto 모드 선택 → "승인" 클릭

### 1-2. KG 빌드 (15초)
> "승인된 온톨로지로 Knowledge Graph를 빌드합니다."

1. "Build KG" 클릭 → 진행률 표시
2. 완료 후 "Q&A" 클릭

### 1-3. Mode A 정형 질의 (30초)
> "정형 데이터에 대한 질의는 Cypher 템플릿으로 밀리초 단위 응답합니다."

1. **Q1**: "고객 김민수가 주문한 상품은?" → 맥북프로, 에어팟프로, 갤럭시탭
   - 배지: `cached`, `rule`, `two_hop`
   - Cypher 경로 펼쳐서 보여주기
2. **Q3**: "가장 많이 팔린 카테고리 Top 3는?" → 노트북(6건), 오디오(4건), 태블릿(2건)

---

## Act 2: 비정형 문서 업로드 (30초)

### 2-1. 문서 관리 페이지 이동
> "이제 비정형 문서를 추가합니다. 제품 설명서와 이용안내를 업로드합니다."

1. 상단 "문서 관리" 보라색 버튼 클릭
2. DocumentsPage 진입

### 2-2. 문서 업로드
1. 3개 파일 선택: `맥북프로_제품설명.md`, `에어팟프로_제품설명.md`, `쇼핑몰_이용안내.md`
2. 업로드 → 처리 중(노랑) → 완료(초록) 자동 갱신
3. 청크 수 확인 (각 문서 2~4개 청크)

> "문서가 자동으로 파싱, 청킹, 임베딩되고 KG와 연결됩니다."

---

## Act 3: Mode C 하이브리드 검색 (2분)

### 3-1. 문서 전용 질의 (40초)
> "Mode C는 문서와 그래프를 동시에 검색합니다. 먼저 문서에만 있는 정보를 질의합니다."

1. "Q&A (Mode C)" 버튼 클릭 → QueryPage 이동
2. 모드 토글 **C 하이브리드** (보라색) 확인

3. **C1**: "맥북프로의 배터리 수명은 얼마나 되나요?"
   - 답변: "최대 22시간(14인치), 24시간(16인치)..."
   - 배지: `hybrid_c`, `hybrid_search`
   - **문서 출처 펼치기** → `맥북프로_제품설명.md`, 관련도 92%

4. **C4**: "반품 정책은 어떻게 되나요?"
   - 답변: "7일 이내 반품 가능, 배송비 6,000원..."
   - 출처: `쇼핑몰_이용안내.md`

> "이 정보는 KG에 없습니다. 문서 벡터 검색으로만 답변 가능합니다."

### 3-2. 하이브리드 질의 (40초)
> "이번엔 KG와 문서를 **결합**해야 답할 수 있는 질문입니다."

5. **C3**: "김민수가 주문한 맥북프로의 주요 사양은?"
   - 답변: KG에서 "김민수→맥북프로" 관계 + 문서에서 사양 정보
   - **서브그래프 컨텍스트 펼치기** → 그래프 경로
   - **문서 출처 펼치기** → `맥북프로_제품설명.md` 사양 청크

> "KG가 '누가 뭘 샀는지' 알려주고, 문서가 '그 제품 사양'을 알려줍니다."

6. **C5**: "애플코리아가 공급하는 제품들의 특징은?"
   - KG: 공급업체→제품 4개 관계
   - 문서: 맥북프로 + 에어팟프로 설명서 각각 인용

### 3-3. Mode 비교 (20초)
> "같은 질문도 모드에 따라 다른 방식으로 답합니다."

7. Mode **A** 토글 → **Q1**: "고객 김민수가 주문한 상품은?"
   - Cypher 기반 정확 답변 (밀리초)
8. Mode **C** 토글 → **C3**: "김민수가 주문한 맥북프로의 주요 사양은?"
   - 그래프 + 문서 결합 답변

> "Mode A는 정확한 정형 질의, Mode C는 비정형 정보를 결합한 심층 답변."

---

## Act 4: 마무리 (30초)

> "정리하면:"

| | Mode A | Mode B | Mode C |
|---|---|---|---|
| 데이터 | KG only | KG subgraph | **KG + 문서** |
| 속도 | 밀리초 | 1~3초 | 1~3초 |
| 답변 | 정확 | 탐색적 | **심층 + 인용** |

> "GraphRAG v5.0은 DDL에서 Knowledge Graph를 만들고, 문서를 업로드하면 자동으로 연결합니다.
> 정형 질의와 비정형 검색을 하나의 플랫폼에서 제공합니다."

---

## 트러블슈팅

| 증상 | 해결 |
|------|------|
| 문서 상태가 "처리중"에서 안 넘어감 | PG 연결 확인, 임베딩 API 키 확인 |
| Mode C 답변이 "관련 문서를 찾을 수 없습니다" | 문서 업로드 후 상태가 "완료"인지 확인 |
| Neo4j 연결 실패 | `docker compose up -d neo4j` 후 재시작 |
| 데모 캐시 미작동 | 질문 텍스트가 정확히 일치해야 함 (데모 버튼 사용) |
