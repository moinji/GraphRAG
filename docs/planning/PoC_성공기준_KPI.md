# PoC DoD / KPI 1장
> GraphRAG Studio — PoC 성공 기준 (2026-02-24 확정)

## 한 줄 목표
> **"DDL 하나로 자동화가 의미 있는지를 숫자로 증명한다"**

---

## KPI 4개 (고정)

| # | KPI | 목표 | 측정 방법 | 실패 기준 |
|---|-----|------|----------|----------|
| 1 | **TTV (Time-to-Value)** | ≤ 10분 | DDL 업로드(T0) → 첫 Q&A 답변(T5) 구간 타이머 | > 15분 |
| 2 | **Human 검토 시간** | ≤ 60분 | Review Mode 진입 → 승인 클릭까지 | > 90분 |
| 3 | **치명 오류율** | ≤ 10% | 골든 50개 중 (방향 반대 + 관계 누락 + 타입 오류) / 50 | > 20% |
| 4 | **데모 성공률** | 4/5 이상 | 데모 질문 5개 중 "답변 + 근거(경로)" 동시 충족 수 | < 3/5 |

---

## TTV 구간 분해

```
T0 ─── DDL 업로드
 │ < 5초 (결정적)
T1 ─── ERDSchema 파싱 완료
 │ < 30초 (FK 규칙 즉시 + LLM 보강 ~20초)
T2 ─── 온톨로지 생성 완료
 │ < 1초 (Auto Mode 자동 승인)
T3 ─── KG 빌드 시작
 │ < 2분 (데모: 미리 적재 + Job 상태 표시)
T4 ─── KG 빌드 완료
 │ < 15초 (템플릿 + Cypher 실행)
T5 ─── 첫 Q&A 답변
───────────────────
     이상적 < 3분 / 보수적 < 10분
```

---

## 데모 질문 5개 + 판정 기준

| # | 질문 | 라우트 | 판정: 답변 정확 | 판정: 근거 출력 |
|---|------|--------|---------------|---------------|
| Q1 | 고객 김민수가 주문한 상품 목록 | cypher_traverse | 상품 이름 3개+ 정확 | Customer→Order→Product 경로 |
| Q2 | 같은 카테고리 리뷰 평점 Top 3 | cypher_traverse | 상품+평점 정확 | Product→Category→Product→Review 경로 |
| Q3 | 가장 많이 팔린 카테고리 Top 3 | cypher_agg | 카테고리+건수 정확 | 집계 Cypher + sample_paths |
| Q4 | 취소 3회+ 고객 공통 구매 상품 | cypher_traverse | 고객+상품 정확 | Order(cancelled)→Customer 경로 |
| Q5 | 쿠폰 사용/미사용 금액 비교 | cypher_agg | 금액 수치 정확 | 집계 결과 + Cypher |

**성공 = "답변 정확" AND "근거 출력" 둘 다 충족**

## PoC 범위 요약

| 포함 | 미포함 (MVP1 이후) |
|------|-------------------|
| DDL(.sql) 업로드 파싱 | 실시간 CDC / DB 직접 연결 |
| FK 규칙 온톨로지 + LLM 보강 | 멀티테넌트 |
| Auto/Review 모드 | 엔터프라이즈 권한/감사 |

| Cypher 템플릿 Q&A (A안) | GraphRAG Global 상시 실행 |
| GraphRAG Local (B안, P1) | Track B 온보딩 챗봇 |
