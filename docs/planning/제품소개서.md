# GraphRAG Studio

### 데이터베이스 설계 파일 하나로 지식그래프를 만들고, 한국어로 질문하는 시스템

---

## 이 문서는 무엇인가요?

이 문서는 GraphRAG Studio가 **무엇이고**, **어떤 문제를 풀고**, **어떻게 동작하는지** 설명합니다.
기술 배경 지식이 없어도 이해할 수 있도록 작성했습니다.

---

## 한 줄 요약

> 데이터베이스 설계 파일(.sql)을 올리면, AI가 데이터 간 연결 관계를 자동으로 파악하고,
> 한국어로 질문하면 데이터에서 답을 찾아주는 시스템입니다.

---

## 어떤 문제를 풀려고 만들었나요?

기업에는 데이터베이스에 많은 데이터가 쌓여 있습니다.
하지만 이 데이터를 꺼내 보려면, 보통 이런 과정을 거칩니다:

1. 궁금한 것이 생김 — 예: "고객 김민수가 뭘 샀지?"
2. 개발자에게 요청
3. 개발자가 SQL(데이터베이스 질의 언어)을 작성
4. 결과를 받음
5. 추가 궁금한 것이 생기면 → 다시 2번부터 반복

단순한 질문인데도 **답을 얻기까지 반나절에서 하루** 이상이 걸립니다.
테이블이 여러 개로 나뉘어 있고, 테이블끼리 어떻게 연결되어 있는지 파악하는 것 자체가 전문 지식을 요구하기 때문입니다.

GraphRAG Studio는 이 과정을 **비전공자 혼자 3분 안에** 끝낼 수 있게 만든 시스템입니다.

---

## 핵심 개념 설명

이 시스템을 이해하려면 두 가지 개념만 알면 됩니다.

### 지식그래프란?

보통 데이터베이스는 엑셀처럼 테이블(표) 여러 개에 데이터가 나뉘어 저장되어 있습니다.
"고객" 표, "주문" 표, "상품" 표가 따로 있고, 사람이 머릿속으로 이 표들을 연결해야 합니다.

**지식그래프**는 이 표들을 **화살표로 연결한 지도**입니다.

```
고객(김민수) ──주문함──→ 주문(ORD-001) ──포함함──→ 상품(노트북 Pro)
                                        ──포함함──→ 상품(무선 마우스)
```

이렇게 연결해 두면 "김민수가 뭘 샀어?"라는 질문에
화살표를 따라가는 것만으로 답을 찾을 수 있습니다.

### 온톨로지란?

지식그래프를 만들기 전에, "어떤 종류의 데이터가 있고, 어떻게 연결되는지"를 먼저 정리해야 합니다.
이 정리한 설계도를 **온톨로지**라고 합니다.

예를 들면:
- **노드(점)**: 고객, 상품, 주문, 카테고리, 리뷰 ...
- **관계(화살표)**: 고객이 주문을 "한다", 주문이 상품을 "포함한다", 리뷰가 상품에 "달린다" ...

GraphRAG Studio는 이 온톨로지를 **SQL 설계 파일에서 자동으로 만들어줍니다.**

---

## 전체 동작 흐름

총 6단계로 구성됩니다. 사용자가 직접 하는 것은 파일 올리기와 버튼 클릭뿐입니다.

---

### 1단계. 설계 파일 올리기

.sql 파일을 선택하고 Upload 버튼을 누릅니다.

.sql 파일은 데이터베이스의 설계도입니다. 어떤 테이블이 있고, 각 테이블에 어떤 항목(컬럼)이 들어가며, 테이블 간에 어떤 연결(FK, 외래 키)이 있는지 적혀 있습니다.

시스템이 이 파일을 분석해서 다음을 자동으로 추출합니다:
- 테이블 목록과 각 테이블의 컬럼 정보
- 테이블 간 연결 관계 (어떤 테이블이 어떤 테이블을 참조하는지)

화면에 "ERD Summary"라는 카드가 나타나며, 테이블 수와 관계 수를 확인할 수 있습니다.

> 소요 시간: 약 5초

---

### 2단계. 온톨로지 자동 생성

"Generate Ontology" 버튼을 클릭합니다.

시스템이 1단계에서 추출한 테이블과 관계 정보를 바탕으로, 지식그래프의 설계도(온톨로지)를 자동으로 만듭니다.

- 테이블 → 노드 타입으로 변환 (예: `customers` 테이블 → `Customer` 노드)
- 외래 키 → 관계 타입으로 변환 (예: `orders.customer_id` → `PLACED` 관계)
- 중간 테이블(조인 테이블) → 관계로 변환 (예: `order_items` 테이블 → `CONTAINS` 관계)

선택 사항으로 "Include LLM enrichment"를 체크하면, AI(Claude)가 관계 이름을 더 자연스럽게 개선하거나, 추가적인 의미 관계를 제안합니다.

> 소요 시간: 약 30초 (AI 보강 시)

---

### 3단계. 검토 및 승인

자동 생성된 온톨로지를 화면에서 확인합니다.

- **Nodes 탭**: 노드 타입 목록 (고객, 상품, 주문 등)
- **Relationships 탭**: 관계 목록 (주문함, 포함함, 속함 등)
- **Changes 탭**: AI가 변경한 내역 (LLM 보강을 사용한 경우)

두 가지 모드가 있습니다:
- **Auto 모드**: 자동 생성 결과를 그대로 승인. 빠르게 진행할 때 사용
- **Review 모드**: 노드나 관계를 직접 수정/추가/삭제한 뒤 승인. 정확도가 중요할 때 사용

수정은 코딩이 아니라, 표의 항목을 클릭해서 내용을 고치는 방식입니다.
확인이 끝나면 "Approve" 버튼을 클릭합니다. 승인 후에는 잠금 처리되어 실수로 수정되는 것을 방지합니다.

> 소요 시간: Auto 1초 / Review 5~10분

---

### 4단계. 데이터 넣기 (선택 사항)

승인이 완료되면 "CSV Data" 섹션이 나타납니다.

CSV 파일(엑셀에서 저장할 수 있는 범용 데이터 형식)로 실제 데이터를 올릴 수 있습니다.
각 CSV 파일은 테이블 이름과 일치해야 하며, 시스템이 컬럼과 데이터 타입을 자동 검증합니다.

**CSV를 올리지 않아도 됩니다.** 이 경우, 시스템이 샘플 데이터를 자동으로 생성하여 다음 단계를 진행합니다. 처음 사용하는 경우 샘플로 먼저 체험하는 것을 권장합니다.

---

### 5단계. 지식그래프 구축

"Build KG" 버튼을 클릭합니다.

시스템이 온톨로지(설계도) + 데이터를 결합하여, 실제 지식그래프를 Neo4j(그래프 전용 데이터베이스)에 구축합니다.

화면에 진행 상태가 실시간으로 표시됩니다:
- **대기(queued)** → **구축 중(running)** → **완료(succeeded)**

완료되면 생성된 노드 수, 관계 수, 소요 시간이 표시됩니다.
문제가 발생하면 자동으로 되돌리고(롤백) 오류 내용을 알려줍니다.

> 소요 시간: 1~2분

---

### 6단계. 한국어 질의응답

구축이 완료되면 "Q&A" 버튼이 나타납니다. 클릭하면 채팅 화면으로 이동합니다.

채팅창에 궁금한 것을 한국어로 입력하면, 시스템이 지식그래프에서 답을 찾아 응답합니다.

미리 준비된 데모 질문 5개(Q1~Q5)를 버튼으로 바로 실행할 수도 있습니다.

**답변에는 근거가 함께 표시됩니다.** 어떤 데이터를 어떤 경로로 연결해서 이 답을 찾았는지 보여주기 때문에, 답변의 정확성을 직접 확인할 수 있습니다.

두 가지 답변 방식이 있습니다:

| 방식 | 동작 원리 | 특징 |
|------|----------|------|
| **A 템플릿** | 미리 정의된 질의 패턴에 매칭하여 실행 | 빠르고 정확함. 추가 AI 비용 없음 |
| **B AI 탐색** | AI가 질문에서 핵심 개체를 추출하고, 그래프를 직접 탐색하여 답변 생성 | 유연하고 다양한 질문에 대응 가능 |

> 소요 시간: 15초 이내

---

### 부가 기능: 그래프 탐색

상단 메뉴의 "Explore Graph" 버튼으로 언제든 접근할 수 있습니다.
지식그래프를 시각적으로 탐색하는 화면으로, 노드를 클릭하면 상세 정보와 연결된 이웃 노드를 확인할 수 있습니다.

## 실제 시연 예시

이커머스(쇼핑몰) 데이터베이스를 사용한 시연 결과입니다.
12개 테이블, 15개 외래 키로 구성된 설계 파일을 사용했습니다.

시스템이 자동으로 인식한 구조:
- **9개 노드 타입**: Customer, Product, Order, Category, Review, Supplier, Address, Coupon, Payment
- **12개 관계 타입**: PLACED, CONTAINS, BELONGS_TO, WROTE, SUPPLIED_BY 등

### 데모 질문과 답변

**Q1. "고객 김민수가 주문한 상품은?"**
> 답변: 노트북 Pro, 무선 마우스, USB-C 허브
> 근거 경로: Customer(김민수) → Order → OrderItem → Product

**Q2. "김민수가 주문한 상품과 같은 카테고리에서 리뷰 평점 Top 3 상품은?"**
> 답변: 상품명 + 평점 목록
> 근거 경로: Product → Category → Product → Review (평점 집계)

**Q3. "가장 많이 팔린 카테고리 Top 3는?"**
> 답변: 카테고리명 + 판매 건수
> 근거 경로: Category ← Product ← OrderItem (건수 집계)

**Q4. "김민수와 이영희가 공통으로 구매한 상품은?"**
> 답변: 공통 구매 상품 목록
> 근거 경로: Customer(김민수) → Order → Product ← Order ← Customer(이영희)

**Q5. "쿠폰 사용 주문과 미사용 주문의 평균 금액 비교"**
> 답변: 쿠폰 사용 평균 금액 / 미사용 평균 금액
> 근거 경로: Order(coupon_used=true/false) → 금액 집계 비교

5개 질문 모두 정확한 답변 + 근거 경로를 출력하여, 데모 성공률 **5/5** 달성.

## 기술 구성 요약

비전공자 입장에서 알아두면 좋은 구성 요소입니다.

| 구성 요소 | 쉽게 말하면 | 역할 |
|----------|-----------|------|
| **FastAPI** (백엔드 서버) | 요청을 받아서 처리하는 두뇌 | 파일 분석, 온톨로지 생성, 질의 처리 등 모든 로직 수행 |
| **React** (프론트엔드) | 사용자가 보는 웹 화면 | 파일 업로드, 검토/수정, 채팅 UI 제공 |
| **Neo4j** (그래프 DB) | 데이터 간 연결 관계를 저장하는 특수 저장소 | 지식그래프의 실제 데이터를 저장하고 질의에 응답 |
| **PostgreSQL** (메타 DB) | 관리 정보를 저장하는 일반 저장소 | 온톨로지 버전, 빌드 이력 등 관리 데이터 보관 |
| **Claude AI** | 자연어를 이해하는 인공지능 | 온톨로지 보강, 질문 분류, B모드 답변 생성에 활용 |
| **Docker** | 설치 없이 실행할 수 있게 묶어주는 포장 | 모든 구성 요소를 한 번에 실행 가능하게 패키징 |

## 현재 상태와 제한 사항

### 현재 가능한 것
- SQL DDL 파일 업로드 및 자동 분석
- FK 기반 온톨로지 자동 생성 + AI 보강(선택)
- 사람이 검토/수정/승인하는 워크플로
- CSV 데이터 업로드 또는 샘플 데이터 자동 생성
- Neo4j 지식그래프 자동 구축
- 한국어 자연어 질의응답 (A 템플릿 / B AI 탐색)
- 그래프 시각적 탐색

### 현재 제한 사항
- 10~15개 테이블, 테이블당 500행 기준으로 최적화 (대용량 미지원)
- 데이터베이스에 직접 연결하지 않음 (파일 업로드 방식만 지원)
- 단일 사용자 환경 (멀티테넌트 미지원)
- 이커머스 도메인에서 가장 많이 검증됨
